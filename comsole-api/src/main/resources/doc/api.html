仿真引擎 API 对接规范 (v7.1 智能全维度版)

📐 核心架构设计说明：
1. 精准容差抓取： 边界条件通过 Box 算法实现空间坐标容差捕获，前端只需传入物理坐标 (x, y, z)。
2. 智能网格： 网格划分全面采用“物理场智能映射”架构。前端只需在对应的物理场下配置 meshConfig。
3. 参数全解耦： 通过 globalParameters 将 UI 上的温度、频段、功率提取为全局变量，前端切换参数时只需修改全局变量。
4. 🌟智能数据路由 (Smart Routing)： 引擎将自动识别 exports 中变量的物理维度。1D 全局变量（如 S参数、频率）将提取为 JSON 数组直出（供 ECharts 渲染）；2D 表面变量（如面热流）将剥离导出至 _2d.vtu；3D 空间变量（如电场、温度）导出至 _3d.vtu，彻底消除前端 ParaView 渲染异常！

一、📋 接口参数详细定义 (Data Dictionary)

1.1 根节点参数 (Root Object)

字段名类型必填说明
stepFilePathString是CAD 模型的服务器绝对路径。如 "E:/temp/mimo.step"
globalParametersObject否全局变量字典。如 "freq_start": "2[GHz]"，后续在属性中可直接以字符串形式引用。
materialsArray是材料配置列表，详见 1.3。
physicsListArray是物理场与边界条件列表（内置专属网格配置），详见 1.4。
multiphysicsListArray否多物理场耦合列表。用于热机、电磁热耦合等。
studyObject否求解类型配置。极速网格预览接口可不传此项。
exportsArray否结果变量导出配置，详见 1.5。极速网格预览接口可不传此项。

1.2 空间抓取器参数 (PointSelectDef)

字段名类型必填说明
entityDimint是几何抓取维度：0 点(Vertex), 1 线(Edge), 2 面(Boundary), 3 体(Domain)。
x, y, zDouble是前端拾取的三维坐标。单位必须是米 (m)。
toleranceDouble否抓取容差(米)。推荐 0.001 以包容 CAD 显示误差。

1.3 材料配置参数 (MaterialDef)

字段名类型必填说明
tagString是材料唯一标识符，如 "mat_aluminum"。
nameString否材料显示名称，如 "纯铝 (Al)"，仅用于 MPH 树状图显示。
entityDimint是赋参维度。主结构填 3，表面薄涂层/介质板填 2。
pointSelectorsArray否空间抓取器。传空数组 [] 代表此材料全局兜底生效。
propertiesObject是材料物理常数键值对。值必须带单位括号。

⚠️ 高危踩坑预警 (底层真名与前端UI映射)：

二、🌐 接口路由说明

2.1 [辅助接口] 空间坐标验算验证 ID

POST /simulation/generic/get-ids

用途：前端用于验证拾取坐标能否准确命中底层的几何面/体。仅读取并解析 CAD 模型，速度较快。

2.2 [前置处理] 极速生成并预览网格

POST /simulation/generic/mesh-preview

用途：仅执行几何导入与智能网格划分，极速返回网格数据的 .vtu 文件供前端渲染网格结构线，不进行耗时的物理场矩阵求解。

2.3 [核心计算] 仿真物理场全自动求解

POST /simulation/generic/run

用途：执行一站式动态多物理场计算。支持 1D/2D/3D 数据分离导出，解决异构数据渲染冲突。
